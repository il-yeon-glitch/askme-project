새로운 기능을 구현해줘: $ARGUMENTS
 
아래 순서를 반드시 따라줘:
 
## 1단계: 계획
먼저 구현 계획을 알려줘. 실행하기 전에 나에게 확인을 받아야 해:
- 바뀌거나 새로 만들어질 파일 목록
- 구현 순서
- 핵심 함수/컴포넌트 이름
- 예상 소요 시간
 
**계획을 보여주고 "진행할까요?"라고 물어봐. 내가 승인할 때까지 코드를 작성하지 마.**
 
## 2단계: 백엔드 구현
- Express 라우트 추가
- 입력값 검증 포함 (CLAUDE.md 규칙 준수)
- 필요하면 DB 스키마 변경
 
## 3단계: 프론트엔드 구현
- React 컴포넌트 추가
- API 연동
- 에러 처리 포함
 
## 4단계: 통합 테스트
- API를 직접 호출해서 정상 동작 확인
- 브라우저에서 확인할 수 있는 방법 안내
 
## 5단계: 변경 요약
- 바뀐 파일과 핵심 변경사항을 요약
- "코드 읽기 체크" 가이드 제공:
  1. 어떤 파일이 바뀌었는가?
  2. 핵심 함수는 무엇인가?
  3. 입력/출력은 무엇인가?
  4. 왜 바뀌었는가?
